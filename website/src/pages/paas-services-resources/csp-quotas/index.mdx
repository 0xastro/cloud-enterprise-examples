In this section, we provide you information on the limits and quotas for the resources we have used in the cloud patterns.

## Virtual Private Cloud(Gen 2)–Quotas and service limits

The  link provided covers  quotas  and  limits  for  IBM  Cloud™  Virtual  Private  Cloud  and  the resources available within it - https://cloud.ibm.com/docs/vpc?topic=vpc-quotas.

## IBM Cloud™ Kubernetes Service 

IBM Cloud™ Kubernetes Service and the Kubernetes open source project come with default service settings and limitations to ensure security, convenience, and basic functionality. Some of the limitations you might be able to change where noted. 
Please refer link for more details - https://cloud.ibm.com/docs/containers?topic=containers-limitations.

## IBM Container registry -Managing quota limits for storage and pull traffic

You can limit the amount of storage and pull traffic that can be used in your IBM Cloud™ account by setting and managing custom quota limits. Please refer link for more details - https://cloud.ibm.com/docs/Registry?topic=registry-registry_quota.
